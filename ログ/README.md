# ログ設計（情報を揮発させない仕組み）

目的: このワークスペース内で起きた「会話・リサーチ・コード・成果物」を、できるだけ揮発させずに残す。

## ディレクトリ構造（初期案）

- `会話ログ/YYYY-MM-DD.md`
  - その日の重要な会話トピックと決定事項の要約
  - 「何を話したか」「どんな方針が決まったか」を残す

- `ログ/セッションログ/`
  - 単発の「まとまりのある回」を1つのセッションとして記録
  - 例: `2026-04-01_プロジェクト整理と学習方針.md`

- `ログ/リサーチ/`
  - リサーチした内容の要約と、参照URL・関連ファイルへのパス
  - 例: `2026-04-01_タスク管理ツール比較.md`

- `ログ/コード/`
  - 生成・編集したコードのスナップショットや場所へのリンク

## ログに必ず残したい情報

- 日付・ざっくりテーマ
- 「何をしたか」種別
  - 会話 / リサーチ / 設計 / 実装 / メモ整理 など
- 成果物がある場合:
  - ファイルパス（例: `プロジェクト/AI講習/カリキュラム.md`）
  - 必要なら簡単なサマリ

## 運用方針（梓側）

- 「これは後で絶対参照しそう」と思う回・作業は、
  - `会話ログ/YYYY-MM-DD.md` にトピックを追記
  - 必要に応じて `ログ/リサーチ/◯◯.md` などに詳細を書く
- 完全な生ログではなく、**後で意味のある単位**でまとめる。
- ローカルフォルダを真のソース・オブ・トゥルースにする（外部サービス連携はその後）。

今後、実際の運用をしながらフォーマットやフォルダ構成は一緒に調整していく。
